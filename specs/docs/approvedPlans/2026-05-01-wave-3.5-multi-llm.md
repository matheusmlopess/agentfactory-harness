# Approved Plan: Wave 3.5 — Multi-LLM Provider Layer
<!-- version: 1.0.0 -->
<!-- approved: 2026-05-01 -->

## Goal

Decouple `agentLoop` from the Anthropic SDK so the harness can run any plan step
against any supported LLM. A thin `LLMAdapter` interface sits between the agent loop
and the provider SDK. `AnthropicAdapter` and `OpenAIAdapter` are the first two
implementations. Provider is selected per-session via `LLM_PROVIDER` env var
(default `anthropic`) and can be overridden per step in `af-plan.json`.

## Reference material

Pattern modelled after `.refs/open-multi-agent/src/llm/` (adapter factory, lazy
imports per provider). Key difference: our adapter interface uses Anthropic
`MessageParam[]` as the canonical internal history format (since Session already
stores it), so adapters own the conversion to their provider's wire format.

---

## LLMAdapter interface

```typescript
// src/core/llm/types.ts

export type Provider = 'anthropic' | 'openai'

// Normalised streaming chunks — both adapters produce these
export type StreamChunk =
  | { type: 'text_delta';       text: string }
  | { type: 'tool_start';       id: string; name: string }
  | { type: 'tool_input_delta'; index: number; json: string }
  | { type: 'message_end';      stop_reason: string }

export interface LLMStreamOptions {
  model: string
  systemPrompt: string
  tools: ToolDef[]
  maxTokens?: number
  signal?: AbortSignal
}

export interface ToolDef {
  name: string
  description: string
  inputSchemaJson: { type: 'object'; properties: Record<string, unknown>; required?: string[] }
}

export interface LLMAdapter {
  readonly provider: Provider
  readonly defaultModel: string
  stream(messages: MessageParam[], opts: LLMStreamOptions): AsyncIterable<StreamChunk>
}
```

`MessageParam` is imported from `@anthropic-ai/sdk/resources/messages.js` — it is
the canonical internal format because Session already uses it.

## Provider defaults

|----------|---------|---------------|
| `anthropic` | `ANTHROPIC_API_KEY` | `claude-opus-4-7` |
| `openai` | `OPENAI_API_KEY` | `gpt-4o` |

Active provider: `LLM_PROVIDER` env var (default `anthropic`).

## agentLoop change

`AgentLoopOptions` gains an optional `adapter` field. When omitted, `createAdapter`
is called with `process.env.LLM_PROVIDER` (default `'anthropic'`). The loop body
replaces the inline Anthropic SDK call with `adapter.stream(...)` and processes
`StreamChunk`s instead of raw Anthropic SDK events.

---

## Schema addition

`StepSchema` gains two optional fields:

```typescript
provider: z.enum(['anthropic', 'openai']).optional(),
model:    z.string().optional(),
```

When set, the executor passes them through to `AgentRunnerFn` which creates a
per-step adapter override. This allows mixing providers in one plan.
